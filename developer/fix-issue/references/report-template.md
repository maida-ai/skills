# Issue Report Template

Create `_ai_report/issue-NUM-DATE.md`, where `NUM` is the GitHub issue number and `DATE` is the current local date in `YYYYMMDD` format.

Keep this generated report untracked or unstaged. Do not include the generated report in commits, commit messages, or committed files.

Use this structure:

```markdown
# Issue NUM Report

Issue: <title>
URL: <issue URL if available>
Resolution: [addressed|need-feedback|skipped]

## Summary

<What was decided and why. Mention if the issue was closed, stale, already fixed, or blocked.>

## Changes

<Detailed explanation of the change. If commits were created, group the explanation by commit and include commit hashes. If no commit was created, say why.>

## Verification

<Commands run and results. State explicitly what was not run.>

## Followups

<Any remaining work, reviewer checks, or user feedback needed. Use "None" if there are no known followups.>
```
